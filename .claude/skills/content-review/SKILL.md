# Content Review — DataSlice

Ревью **контента** уроков: понятен ли урок новичку. Дизайн — отдельный скилл `design-review`.
Этот скилл отвечает на один вопрос по каждому уроку:

> **Позволяет ли урок человеку, ничего не понимающему в теме, разобраться в ней
> на уровне, достаточном для выполнения простых задач junior/middle аналитика данных?**

Это верховный критерий (задан владельцем). Всё ниже — инструменты его достижения.

## Область (две разные модели контента)
- **Статистика** (`/stats`, `src/content/lessons/*.json`) — 56 уроков в beats-модели:
  `intro` + `beats[]` (predict/reveal) + `definitions[]` (с `simple`) + `practice`/`realLife`.
- **Иерархии метрик** (`/metrics`, `src/content/industries/*.json`) — 15 индустрий, модель
  ДРУГАЯ: дерево метрик `root` → `children` с полями `title`/`formula`/`note`, плюс `pyramid`
  (бизнес/продукт/операционные) + глоссарий (`glossary.js`). Контент здесь = поля `note`
  (пояснения к каждой метрике) и структура дерева. Критерий тот же: поймёт ли новичок, что
  это за метрика, зачем она и как связана с North Star — на уровне junior/middle.
  novice-test для metrics: сможет ли новичок по дереву объяснить, из чего складывается
  North Star индустрии и какую метрику он как аналитик стал бы двигать.

## Две линзы
- `rules/clarity.md` — **верховная**: объяснительная ясность, достаточность, novice-test.
- `rules/copy-voice.md` — тон, авторский голос, анти-ИИ.

## Эталон-референсы (вынесены в `reference/`, НЕ зависят от внешних URL и памяти)
- `reference/sfda-priyomy.md` — конкретные приёмы понятности из учебника SFDA2022 (А. Гардт),
  разобранного по главам. Это источник правды по тону и структуре объяснения.
- Сквозной пример курса — пиццерия «Мама Джонс».

## Пайплайн (единица = урок; коммит = МОДУЛЬ)

1. **Прочитать урок** — JSON из `src/content/lessons/<slug>.json` (поля `intro`, `beats[]`,
   `definitions[]`, `practice`, `realLife`). Для рендера — открыть `/stats/<slug>` (dev-сервер).
2. **novice-test** (ядро) — прочитать урок, НЕ зная темы, отметить первую точку потери понимания.
   Финальный вопрос: сможет ли новичок объяснить концепцию своими словами и решить простую
   задачу уровня junior/middle по ней?
3. **Прогнать линзы** `clarity.md` + `copy-voice.md` по сигналам, сверяясь с `reference/sfda-priyomy.md`.
4. **Править по достаточности** (не по объёму): дописать РОВНО недостающий механизм там, где
   новичок теряется; резать воду. `simple` у технических определений; сквозной пример.
5. **Коммит на МОДУЛЬ** (не на урок): один коммит на все правки модуля. Отметить модуль в
   `reports/goal-progress.md`.

## Границы
- Правится текст уроков с суждением (это авторский голос — держать тон SFDA-манеры).
- Структурные решения (подключить/удалить урок, состав курса) — эскалировать владельцу.
- Дизайн (вёрстка, графики, вьюпорты) — НЕ здесь, это `design-review`.

## Критерий готовности модуля
Все уроки модуля прошли novice-test на пороге «достаточно для простых задач junior/middle»:
для каждого либо подтверждено, что порог достигнут, либо пробел исправлен. Модуль [x] в goal-progress.
