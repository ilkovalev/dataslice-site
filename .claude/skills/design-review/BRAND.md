# BRAND.md — эталон намерения DataSlice

Это **тонкий указатель на намерение**, а не описание текущего вида сайта и не полный стайлгайд.
Оценку качества несут `rules/` (внешняя планка). Этот файл отвечает только на вопрос
«с каким замыслом сверять» и «что здесь намеренная константа».

> Правило `rules/visual-design.md#ai-default-looks` прогоняется **и по этому файлу тоже**:
> если само намерение шаблонно — это находка, а не эталон.

---

## 1. Намерение (источник правды — вне этого репозитория)

Не дублируем, ссылаемся:

- **Аудитория, тон бренда, цель канала** → `Obsidian/Аналитика/Контент/Стратегия/Контент_стратегия.md`
  Кратко: русскоязычные новички в аналитике; тон — авторский практик, практично, без воды.
- **Роль сайта + целевое действие** → `README.md` этого репозитория.
  Сайт = **лид-магнит для Telegram-канала DataSlice**. Воронка: посетитель проходит
  интерактивный урок → чувствует ценность → **подписывается на канал** (`t.me/dataslice`).
  В коде это измеримо: `track('tg_click')`, кнопка TG в шапке и подвале.
- **Тон и формат уроков** → память `feedback-dataslice-course-tone` + референсы
  SFDA2022 (тон) и Seeing Theory (интерактив). Оба сошлись на едином ядре понятности —
  см. `rules/clarity.md`.

**Верховная цель, которой подчинено всё:** урок должен объяснить концепцию так,
чтобы понял непогружённый человек или ребёнок. Не только график — весь урок.

---

## 2. Целевое действие (для UX-линзы)

Единственное действие, к которому ведёт страница: **подписка на Telegram-канал**
после того, как урок принёс пользу. Всё остальное (навигация по урокам, метрики,
глоссарий) — поддержка этого пути. Линза `rules/ux-blocks.md` судит каждый блок
вопросом «приближает ли он посетителя к этому действию или уводит».

---

## 3. Визуальные токены (ЧЕРНОВИК — подняты из кода, требуют подтверждения)

Значения взяты из `tailwind.config.js` и `src/index.css`. Это **не закон**: по каждому
токену владелец подтверждает «намеренно» или «случайно так вышло — оценивать с нуля».
До подтверждения `rules/` считают выброс за пределы этих значений находкой уровня «полиш».

### Цвет
| Токен | Значение | Роль (как в коде) |
|---|---|---|
| фон страницы | `#FFF8EF` | тёплый бежевый, `body` |
| текст | `#20242e` | основной |
| `ink` | `#f3ebdd` | ⚠️ имя «ink», но значение светлое — уточнить семантику |
| `panel` | `#ffffff` | панели/карточки |
| `accent` / `brand` | `#2ab8eb` / `#29abe2` | голубой акцент |
| `cyanink` | `#0a6c97` | тёмный акцент для текста/кнопок; заявлен ≥4.5:1 на бежевом (WCAG AA) |
| фокус | `2px solid #0a6c97`, offset 2 | клавиатурный фокус |

### Типографика
- **Единственное семейство:** `Golos Text` (вариативный 400–700, самохостинг, кириллица+латиница).
  Шкалу размеров зафиксировать здесь после ревью (сейчас в коде — Tailwind-классы, не токены).

### Тема и движение
- **Только светлая тема** (`color-scheme: light`). Тёмной темы нет —
  правила НЕ требуют dark mode; `chart-legibility` проверяет только светлую.
- Анимации обязаны уважать `prefers-reduced-motion` (в коде уже так). Движение — только осмысленное
  (сходимость, отклик на действие), не декоративное. См. `rules/dataviz.md#sim-convergence`.

### Локали
- Двуязычие RU/EN (`/stats/...` и `/en/stats/...`). Правила проверяют **обе** локали;
  сквозной пример не должен теряться в переводе (`Мама Джонс` → `Mama John's`).

## 4. Сквозной пример

Каноничный якорь курса — пиццерия **«Мама Джонс»** (🍕 в бренде, сквозная в `center-measures`).
Требование — в `rules/clarity.md#worked-example`: пример должен **доказывать** мысль,
а не украшать, и проходить урок насквозь (intro → биты → «что это значит»).
