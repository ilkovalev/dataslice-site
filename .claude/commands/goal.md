Запусти пайплайн скилла **design-review** для: `$ARGUMENTS`

Скилл — `.claude/skills/design-review/SKILL.md` (движок: правила + линзы + capture).
Верховный критерий над всеми линзами — **достигает ли урок своей цели: понять концепцию
способен непогружённый человек/ребёнок** (`rules/clarity.md`). Отсюда имя `/goal`.

## Если аргумент — `all` (скан всего курса)
1. Убедись, что dev-сервер поднят (`http://localhost:4203`), иначе подними: `npm run dev -- --port 4203 --strictPort` (в фоне).
2. Контентный скан по JSON (надёжный, без сервера): для каждого урока — определения без `simple`, «половинчатые» predict/reveal, потеря сквозного примера в EN.
3. Поведенческий скан: `node .claude/skills/design-review/scripts/scan-all.mjs` — ошибки консоли (сломанные виджеты), overflow, безымянные графики.
4. Сведи в `reports/Скан_<дата>.md`: приоритизированный список уроков с реальными сигналами. **Только отчёт, без правок.**

## Если аргумент — slug урока (полное ревью одного)
1. Захват: `node .claude/skills/design-review/scripts/capture.mjs <slug>` → скрины (3 вьюпорта × RU/EN) + `metrics.json`.
2. Прочитай сам урок: `src/content/lessons/<slug>.json` (поля `intro`, `beats`, `definitions`).
3. Прогони линзы по `rules/`, **начиная с `clarity.md`** — включая проход `novice-test`
   (читать урок, не зная темы, отметить первую точку потери понимания). Остальные линзы —
   `copy-voice`, `visual-design`, `dataviz`, `interactivity`, `ux-blocks`, `accessibility`.
4. **Каждый инструментальный кандидат проверяй глазом** по скриншоту — контраст-детектор
   врёт на полупрозрачных фонах (`bg-*/NN`).
5. Сведи в `reports/Отчёт_<slug>_<дата>.md` по шаблону из SKILL.md: severity (blocker/важное/полиш),
   привязка к скрину/полю, конкретная правка + раздел «что хорошо».

## Границы
По умолчанию выход — **отчёт, не авто-правки**. Если пользователь просил править —
безопасное (a11y, визуал-полиш) можно сразу; текст уроков (формулировки, `simple`,
структура) — только с его подтверждением, отдельным коммитом на урок, т.к. это авторский голос.
