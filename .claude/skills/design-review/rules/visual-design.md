# Линза: Визуальный дизайн

Вход: скриншоты (все вьюпорты) + метрики `capture` (реальные `font-size`/`weight`/`line-height`,
отступы, цвета). Часть сигналов инструментальна (числа), часть — суждение по картинке.
Содержание опирается на skill `frontend-design`. Severity по умолчанию — «важное».

---

## visual-hierarchy — визуальная иерархия
Куда падает глаз первым — должно совпадать с самым важным на экране.
- **Вход:** скриншот (чистое суждение).
- Сигналы: главный элемент экрана визуально не выделен; несколько элементов борются за первичное внимание; порядок чтения не ведёт к цели урока.

## typography-scale — шкала шрифтов
Шкала — главный носитель иерархии. Каша размеров → новичок не понимает, что главное.
- **Вход:** гистограмма реальных `font-size`/`weight` из `capture`.
- Сигналы: > 5 уникальных `font-size` на первом экране или значения вне шкалы `BRAND.md`; заголовок и body неразличимы (Δsize < 4px **и** одинаковый weight).

## prose-line-height — интерлиньяж прозы (Golos Text)
- **Вход:** `tightProse` из `capture` (абзацы с ratio lineHeight/fontSize < 1.55).
- **Сигнал:** ratio < 1.75 на длинных абзацах. ВАЖНО: Golos Text имеет крупный x-height,
  поэтому стандартные 1.5–1.63 (`leading-relaxed`) читаются ПЛОТНО. Целевой ratio ≥ 1.75–1.8.
- Найдено 2026-07-10: вся проза была на `leading-relaxed` (1.63) → выглядела сжато (замечено
  владельцем). Поднято глобально до `leading-[1.8]` в BeatsLesson. capture теперь мерит это
  (раньше НЕ мерил — прямой пробел ревью, из-за него пропущено).

## spacing-rhythm — ритм отступов
Единый ритм вертикальных отступов = ощущение порядка.
- **Вход:** измеренные отступы между блоками из `capture`.
- Сигналы: отступы не сводятся к единой шкале (произвольные значения); одинаковые по смыслу блоки разделены по-разному; слипшиеся или, наоборот, разорванные группы.

## ai-default-looks — избегать AI-дефолтных решений
Прогоняется по странице **и по `BRAND.md`**. Черный список из `frontend-design`:
крем `#F4F1EA`+высококонтрастный serif+терракота; near-black+кислотный акцент; broadsheet с hairline-линейками и нулевым радиусом.
- Сигнал: визуальное решение читается как дефолт «для любого сайта», а не выбор под DataSlice. Структурные приёмы (нумерация 01/02/03, дивайдеры) применены декоративно, а не кодируют смысл.
- Примечание: фон DataSlice тёплый бежевый `#FFF8EF` — проверить, что это осознанный выбор, а не дефолт №1.
