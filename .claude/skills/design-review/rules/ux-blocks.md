# Линза: UX и смысл блоков

Вход: скриншот **всей** страницы (композиция целиком) + целевое действие из `BRAND.md#2`.
Судит не отдельный элемент, а роль каждого блока в пути к цели. Severity по умолчанию — «важное».

---

## block-purpose — у каждого блока есть работа на цель
Целевое действие сайта — подписка на TG-канал после пользы от урока (`BRAND.md#2`).
- Сигналы: блок не приближает ни к пониманию урока, ни к целевому действию — кандидат на удаление; два блока делают одну работу; блок красив, но пуст по функции.
- Тест каждого блока: «что сломается для посетителя, если его убрать?» Нет ответа → находка.

## path-to-action — путь к целевому действию читается
- Сигналы: пройдя урок, посетитель не встречает приглашения подписаться в осмысленный момент; CTA на TG есть, но оторван от ценности («подпишись» без «вот что ты получишь»); призыв повторяется навязчиво до того, как дана польза.

## page-job — у страницы одна ясная задача
Каждая страница (`/stats`, `/stats/:slug`, `/metrics`, `/glossary`) делает одну работу.
- Сигналы: страница пытается решать несколько задач сразу; посетитель не понимает, где он и что делать дальше; заголовок/первый экран не сообщают задачу страницы.

## content-density — плотность под аудиторию-новичка
- Сигналы: стена текста/формул до первого интерактива; или, наоборот, пустой экран без ориентиров; когнитивная нагрузка первого экрана слишком высока для непогружённого.
